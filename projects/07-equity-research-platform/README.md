# Autonomous Equity Research Platform

A multi-agent pipeline (built on Claude Code subagents) that turns a single stock ticker into a
fact-checked, brand-themed equity research PDF — the same kind of report a sell-side analyst would
produce, generated end-to-end with no manual data entry.

**Sample output:** [`data/INFY/INFY_Equity_Research_Report.pdf`](data/INFY/INFY_Equity_Research_Report.pdf)
— a real report generated by this pipeline for Infosys (INFY), unedited.

## Why this isn't "just an LLM call"

A single prompt asking an LLM to "write an equity research report" will hallucinate numbers and
silently launder bad data into confident-sounding prose. This pipeline is built around three
specific guardrails instead:

- **Valuation is computed in Python, not by the LLM.** DCF, WACC, and CAPM cost of equity are
  calculated in deterministic code (`src/valuation.py`), never estimated in natural language. This
  was a deliberate response to a bug from an earlier project in this portfolio, where an
  LLM-adjacent spreadsheet model compared annualized returns against monthly volatility — the kind
  of unit-mismatch error that's invisible in prose but obvious in code.
- **The writer can't write files, and can't compute.** The `report-writer` agent is scoped to
  `Read` only. Every number it cites must trace back to one of three upstream JSON artifacts
  (validated financials, industry research, valuation) — it has no tool access that would let it
  invent or recompute a figure.
- **A separate agent fact-checks the draft before it ships.** `narrative-fact-checker` cross-reads
  every claim in the drafted report against the source data and flags anything unverifiable. Flags
  route back to the writer for a correction pass (up to 2 retries); anything still unresolved ships
  with an explicit `[UNVERIFIED]` marker rather than being silently dropped.

## Pipeline

```
screener-scraper          → raw_financials.json
financial-data-validator  → validated_financials.json
industry-research         → industry_notes.json   (runs alongside the two stages above)
valuation-engine          → valuation.json          (DCF / WACC / CAPM, computed in Python)
report-writer              → report draft (Markdown, returned as text — not written to disk)
narrative-fact-checker      → factcheck_report.json  (loops back to report-writer on flags)
report-pdf-builder           → <TICKER>_Equity_Research_Report.pdf
```

Each stage reads/writes its own artifact under `data/<ticker>/` rather than passing data through
chat context — this keeps large financial datasets out of the model's context window and makes
every stage independently re-runnable without re-doing earlier work.

Full agent-by-agent contract and coordination details: [`.claude/agents/WORKFLOW.md`](.claude/agents/WORKFLOW.md).

## Report contents

Cover page (rating, current price, DCF fair value, upside/downside) → executive summary & thesis →
industry overview → financial analysis (revenue/margin/cash-flow trends) → DCF valuation bridge →
key risks → disclaimer. Rendered as a single HTML+CSS document (WeasyPrint) with matplotlib charts
at 200 DPI, themed to the subject company's actual brand colors.

The PDF-rendering logic itself was generalized into a separate, reusable Claude Code skill
(`pdf-report-builder`, domain-agnostic — not specific to equity research) so the same
chart/layout/theming building blocks can be reused for other kinds of branded report across other
projects.

## Running it

This isn't a standalone app — there's no `requirements.txt` or web server. It runs inside a Claude
Code session: invoking the `research-orchestrator` subagent for a ticker (e.g. "generate an equity
research report for INFY") sequences all seven specialist subagents above and produces the PDF.

## Status

Validated end-to-end on a real ticker (Infosys/INFY) — scraped, validated, valued, drafted,
fact-checked, and rendered without manual intervention. See [`CLAUDE.md`](CLAUDE.md) for the full
build history, including real bugs found and fixed along the way (a WeasyPrint flexbox bug that
silently drops chart images, a hardcoded growth-rate figure caught by the fact-checker, and
screener.in's lack of historical ROE/D-E series being treated as an expected data gap rather than a
scraping failure).
